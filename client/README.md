# `client` Folder

## 📌 What does this folder do?

This folder contains the **FrontEnd** code of the application, that is, the interface that end users interact with.

Here, the screens and components are developed that allow:

* 👤 User registration and login.
* 📍 Visualization of nearby markets and restaurants based on location.
* 🛒 Catalog of available food items about to expire.
* 🔔 Reception of smart notifications about products of interest.
* 💳 Interaction with payment gateway and receipt generation.

The code is developed following best practices, using **JSDoc** documentation to maintain a clear, readable, and easily scalable structure.

---

## ⚙ Installation

1. **Clone the repository** (if you don't have it yet):
```bash
git clone https://github.com/restrepo0510/equipo-3-ings-202520.git
```

2. **Navigate to the client folder:**
```bash
cd client
```

3. **Install the required dependencies:**
```bash
npm install
```

4. **Run the project in development mode:**
```bash
npx expo start
```

In the console, you will see options to open the app on:

* 💻 Development build
* 🤖 Android emulator
* 🍎 iOS simulator
* 🚀 Expo Go, a lightweight option to test the app.

You can start developing by editing the files inside the `app/` directory, which uses **file-based routing** to organize views.

---

## 🧩 Structure and Standards

The project follows these standards:

* **Main language:** JavaScript
* **Framework:** React Native with Expo
* **Documentation:** JSDoc to describe functions, components, and props
* **Code style:** Based on ESLint and Prettier
* **Best practices:** Modular, reusable, and commented code to facilitate maintenance

---

## 🔄 Reset the project from scratch

If you want to reset the project to its initial state, run:
```bash
npm run reset-project
```

This will move the example code to the `app-example/` folder and create a new empty `app/` folder to start from scratch.
