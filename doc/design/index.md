## 📂 Archivos y Propósito

- **ux.md** → Contiene los **wireframes** de las pantallas principales del producto, junto con una URL al diseño completo. Cada pantalla se nombra de acuerdo con su funcionalidad (ejemplo: `/home`, `/inventario`, `/registro`).  
- **ui.md** → Incluye los **diseños finales** de las pantallas, junto con la URL al prototipo. Se muestra la evolución desde los wireframes hacia un diseño consistente, accesible y con guía de estilos definida.  
- **diagrams.md** → Presenta los **diagramas UML y de arquitectura** que ilustran el comportamiento y diseño del software:  
  - **Diagrama Ad hoc**: visión general simplificada del sistema, útil para entender rápidamente su alcance.  
  - **Casos de uso**: muestra cómo los diferentes actores interactúan con el sistema y qué funcionalidades principales cubre el producto.  
  - **Secuencia**: detalla paso a paso la interacción entre actores, componentes y objetos en cada caso de uso.  
  - **Modelo relacional**: describe la estructura de la base de datos y las relaciones entre entidades.  
  - **Componentes**: explica cómo se organizan los módulos del sistema y sus interacciones a nivel de arquitectura.  
- **fe.md** → Documento con el análisis de **frameworks frontend** evaluados. Incluye ventajas, desventajas y justificación de la elección final para construir la interfaz de usuario.  
- **be.md** → Documento con el análisis de **frameworks backend** evaluados. Se comparan diferentes opciones y se justifica la decisión tomada para la capa de negocio y servicios.  
- **index.md** → Este archivo. Sirve como **punto de entrada** a toda la documentación de diseño, explicando la estructura y brindando enlaces de navegación.
